# laiue

Воксельный каркас под Windows без CRT: крошечный лаунчер `laiue.exe`
загружает `laiue_core.dll`, ядро создаёт окно, подключает raw input,
генерирует мир чанками и рисует его через Direct3D 12.
В комплект входит отдельный authoritative `laiue_server.exe`; текущий
безопасный вертикальный срез мультиплеера работает только через loopback.

## Сборка

Требования: любая Visual Studio с C-тулчейном (CMake сам берёт самую новую
установленную), CMake ≥ 3.28. Для ninja-пресетов — Ninja,
для clang-пресетов — LLVM (clang-cl + lld-link).

Перед повторной сборкой закройте клиент и сервер из соответствующего
`build/.../bin/<Configuration>`: Windows не позволяет заменить загруженные
DLL. CMake автоматически остановит сборку раньше линкера и покажет PID
блокирующего процесса. Подробности и команды диагностики находятся в
[CONTRIBUTING.md](CONTRIBUTING.md#перед-каждой-пересборкой).

```cmd
:: последняя установленная Visual Studio
cmake --preset visual-studio
cmake --build --preset visual-studio-release

:: Ninja + MSVC (из Developer Command Prompt)
cmake --preset ninja-msvc-release
cmake --build --preset ninja-msvc-release

:: Ninja + clang-cl
cmake --preset ninja-clang-release
cmake --build --preset ninja-clang-release
```

## Архитектура

```
laiue.exe                 — лаунчер без CRT, только LoadLibrary + Start
└── laiue_core.dll        — лёгкая UI-оболочка и жизненный цикл сессии
    ├── laiue_window.dll  — окно, цикл сообщений, mouse look, время
    ├── laiue_input.dll   — raw input: клавиатура и мышь
    ├── laiue_world.dll   — воксельный мир: шум, чанки-дельты, кеш высот
    ├── laiue_mesher.dll  — бинарный greedy meshing (мир -> геометрия)
    ├── laiue_render.dll  — Direct3D 12, GPU-резидентные меши
    ├── laiue_content.dll — каталог, форматы и проверяемые bundle
    ├── laiue_mod.dll     — client/server роли и хост нативных модов
    ├── laiue_network.dll — bounded protocol и client/server transport
    ├── laiue_physics.dll — AABB и столкновения с вокселями
    ├── laiue_gameplay.dll — движение, стойка и value-модель инвентаря
    └── laiue_interaction.dll — raycast и редактирование блоков

laiue_server.exe          — headless composition root без renderer/UI/core
├── laiue_network.dll     — соединения, framing, очереди и rate limits
├── laiue_content.dll     — серверный bundle модов/шейдеров/текстур
├── laiue_mod.dll         — server/both моды из server_enabled.txt
├── laiue_world.dll       — авторитетный мир
├── laiue_gameplay.dll    — авторитетная физика игрока
└── laiue_interaction.dll — проверенная сервером правка блоков
```

Все модули собираются без CRT (`/NODEFAULTLIB`), DLL — без точки входа.
Профиль максимальной производительности использует `/GS-` и не включает
CFG/CET; ASLR и DEP/NX сохранены как не добавляющие инструкций в горячий путь.
Состояние — непрозрачные экземпляры (`Window*`, `Input*`, `World*`,
`Renderer*`), глобальных переменных нет. Рендерер не знает о мире,
мир не знает о геометрии; их связывает модуль mesher.

Конвейер чанков: пул рабочих потоков строит меши бинарным greedy meshing
(колонна чанка 64 блока = один uint64, слияние граней — битовые маски),
квад вместе с материалом упакован в 8 байт и разворачивается вершинным
шейдером по SV_VertexID (vertex pulling — без вершинных и индексных
буферов), а пиксельный шейдер берёт материал из texture array. Меши
суб-аллоцируются из общих DEFAULT-буферов GPU (staging-загрузка,
отложенное освобождение под fence), рисуются спереди-назад с отсечением
по пирамиде видимости. Загрузка — с бюджетом на кадр и гистерезисом
радиуса. Origin rebasing: камера двигается в double, рендер работает
в координатах относительно неё — мир не деградирует вплоть до пределов
int64 (±9,2·10¹⁸ блоков).

Расширение HUD по `F3` показывает очередь meshing, среднее время сборки,
отменённые результаты, draw calls, объём upload и заполнение GPU-пула.
Загрузка мешей использует постоянный mapped upload-ring по кадрам,
не создавая отдельный D3D12 upload-resource для каждого обычного чанка.
Главное меню создаёт только окно, swapchain, UI и один статичный PNG-фон:
мир, meshing workers, физика, DLL-хост модов и block texture array лениво
создаются после выбора мира/допуска сервером и освобождаются при возврате.

Новый модуль объявляется в CMake одной командой:

```cmake
laiue_add_module(имя SOURCES <файлы...> [LINK <библиотеки...>])
```

Подробные границы и направление зависимостей описаны в
[docs/architecture.md](docs/architecture.md), а план технических
улучшений — в [docs/improvement_plan.md](docs/improvement_plan.md).
Правила разработки и локальные проверки находятся в
[CONTRIBUTING.md](CONTRIBUTING.md).
Корневая сборка автоматически отклоняет запрещённые межмодульные include.

Локальный сервер запускается из каталога сборки:

```powershell
./laiue_server.exe
./laiue.exe
```

В клиенте выберите «Сетевая игра»; автоматического подключения при старте
нет. Сервер слушает только `127.0.0.1:27180`, настройки находятся в
`server.cfg`, список клиента — в `servers.txt`. До допуска сверяется точный
server/both-набор модов, а разрешённая сервером загрузка `.lmp/.lsp/.ltp`
требует явного подтверждения и проверяется SHA-256. Ограничения текущей реализации, threat model и обязательный план
перехода на MsQuic/TLS 1.3 описаны в
[docs/multiplayer.md](docs/multiplayer.md). Открывать текущий plaintext TCP
в LAN или Internet нельзя.

## Управление

- главное меню — одиночная игра со слотами миров или сетевая игра со списком серверов
- `Esc` — назад/выход в главном меню; в игре — пауза, возврат в главное меню или выход
- `G` — переключить креатив/выживание
- `E` — открыть/закрыть инвентарь на 36 слотов
- `1`…`9` или колесо мыши — выбрать ячейку хотбара
- в выживании: `W A S D` — 4 блока/с, `Ctrl` — 6 блоков/с,
  `Shift` — плавно присесть, удерживаемый `Space` — автоматические прыжки;
  прыжок во время бега даёт горизонтальный импульс до 8 блоков/с
- в креативе: `W A S D` — полёт, `Space` — вверх
- `F7` — включить mouse look, `Shift+F7` — выключить
- `V` — переключить вертикальную синхронизацию
- `F3` — показать/скрыть расширенную диагностику стриминга и рендера
- `ЛКМ` — в выживании удерживать до заполнения индикатора; выпадает предмет и появляются частицы
- `ПКМ` — поставить выбранный блок; в выживании расходуется один предмет

В креативе блок ломается сразу, установка не расходует предметы, а дропы,
индикатор ломания и частицы полностью отключены. Дропы и частицы рисуются
пакетно через GPU instancing, без выделений памяти в кадре.

Точные ускорения, трение и времена анимации описаны в
[docs/player_physics.md](docs/player_physics.md).

## Пользовательское содержимое

Единая система различает моды `.lm/.lmp`, шейдеры `.ls/.lsp` и текстуры
`.lt/.ltp`. Категории `.lr/.lrp` и `.ld/.ldp` удалены как не
имевшие runtime-потребителей. Форматы не подменяют
друг друга; каталоги и правила описаны в
[docs/content_formats.md](docs/content_formats.md). Целевая архитектура
подключаемого содержимого — какие категории остаются, как устроены моды
и контракт шейдерпаков — в
[docs/content_architecture.md](docs/content_architecture.md).

Моды — нативные DLL в каталогах `mods/<имя>.lmp` (манифест `mod.lm` +
DLL). Манифест задаёт стабильный `id` и сторону `client`, `server` или
`both`; клиентский порядок хранит `enabled.txt`, серверный —
`server_enabled.txt`. Мод не линкуется с игрой: функции движка приходят таблицей
`LaiueModApi`, поэтому мод пишется и отлаживается без исходников — нужен
только `sdk/laiue_mod_api.h` рядом с исполняемым файлом. Включение —
тумблером на вкладке «Настройки → Моды» (загрузка и выгрузка без
перезапуска), порядок хранит `mods/enabled.txt`. Моды-библиотеки
публикуют интерфейсы для других модов (`publishInterface` /
`queryInterface`). В комплекте пять примеров, включая пару
«библиотека + потребитель». Версия игры видна внизу меню паузы; мод
требует минимальную через `game = MAJOR.MINOR`. Всё — в
[docs/modding.md](docs/modding.md).

Текстурпаки `.ltp` загружаются из каталога `textures` рядом с исполняемым
файлом. Формат LTP и сборка исходных PNG описаны в
[docs/texturepacks.md](docs/texturepacks.md).

Шейдерпаки `.lsp` подключаются в меню «Настройки → Паки» или через
`shaders/active.txt`. В комплекте идут `photon.lsp` (атмосферный рендер с
туманом и тонемаппингом) и `vibrant.lsp` (яркий чистый вид). Меню настроек
прокручивается колесом мыши и работает на любом разрешении; там же —
переключатель полного экрана и кнопки возврата к встроенным шейдерам и
текстурам.

## Сохранения

Каждый мир сохраняется в своём каталоге `saves/<слот>` автоматически при выходе и кнопкой
«Сохранить мир» (Настройки → Админ): seed, правки блоков с координатами
произвольной точности, время суток, позиция и режим игрока, состав
модов (`mods.lock`), инвентарь (`inventory.dat`) и данные модов (`moddata/`).
Загрузка выполняется после выбора слота;
битое сохранение не роняет игру. Формат —
[docs/world_format.md](docs/world_format.md).

## Время, свет и текстуры

Игровые сутки (по умолчанию за 60 минут, старт в 9:00): солнце едет по
небу, рассветы и закаты подкрашивают небо и свет, ночью мир освещает
луна. Свет пишется в линейном пространстве и кодируется в sRGB на
выводе. Часы — в оверлее и в настройках.

Настройки (Esc → Настройки) разбиты на разделы: «Графика» (поле зрения,
рендер, вертикальная синхронизация), «Админ» (время суток, скорость
времени: пауза/обычная/быстрая/реальное время), «Управление»
(чувствительность мыши, скорость полёта). Всё применяется сразу.

Текстуры — собственные, процедурные: `tools/generate_textures.ps1`
детерминированно печёт бесшовные пиксель-арт 32x32 (земля, верх и бок
травы) и карты нормалей с ambient occlusion в альфе. Пак собирается в
формат LTP2 (`build_texture_pack.ps1 -IncludeNormals`); шейдер чанков
освещает нормали солнцем по касательному базису грани.

## Широкоугольный рендер

В настройках (Esc → Настройки) — ползунок поля зрения 0–360° и выбор
рендера: «Авто», «Перспектива», «Рыбий глаз», «Панорама (цилиндр)».
Классическая перспектива геометрически не способна на 180° и выше, поэтому
широкий угол рисуется через кубмапу: сцена растеризуется на грани куба,
ориентированного по осям вида, а полноэкранный проход разворачивает её
в выбранную проекцию (равноудалённый рыбий глаз или цилиндр).

Стоимость держится на уровне обычного рендера: активны только грани,
в которые смотрит проекция; на каждой грани рисуется и очищается только
задействованный прямоугольник (off-center пирамида + viewport/scissor,
своё отсечение по пирамиде на каждую грань); разрешение граней подбирается
под плотность пикселей экрана и уменьшается с ростом угла. Прямоугольники
и проекции пересчитываются только при смене поля зрения, размера окна или
проекции. Режим «Авто» до 120° использует обычный однопроходный рендер
без каких-либо накладных расходов, выше — переключается на рыбий глаз.

Меню рисуется своим слоем: квады с vertex pulling, скругления — SDF
в пиксельном шейдере, текст — атлас глифов, запечённый из Segoe UI через
GDI при старте и при смене масштаба окна.
