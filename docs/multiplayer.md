# Мультиплеер

Версия 0.5.0 содержит локальный client/server vertical slice:

- `laiue_server.exe` — отдельный headless authoritative server;
- `laiue_network.dll` — общий bounded TCP transport, протокол v4;
- сервер считает физику с частотой 60 Гц и отправляет состояние с частотой
  20 Гц;
- клиент отправляет намерения, предсказывает своё движение и выполняет
  reconciliation;
- сервер владеет блоками, инвентарём, дропами и подбором предметов.

Listener намеренно принимает только `127.0.0.1`. Полного initial world
snapshot, interest management и отображения удалённых игроков пока нет.
Сервер сохраняет дельты мира и данные модов, но не состояние игроков и их
инвентари. Это фундамент локальной разработки, не готовый Internet transport.

## Запуск

```powershell
cmake --preset ninja-clang-release
cmake --build --preset ninja-clang-release --parallel
cd build/ninja-clang-release/bin/Release
./laiue_server.exe
./laiue.exe
```

Клиент подключается через «Сетевая игра». Список находится в `servers.txt`.
Настройки сервера — в `server.cfg`:

```text
port = 27180
maximum_peers = 16
world_seed = 42
allow_content_downloads = false
```

Порт и загрузки можно переопределить через `LAIUE_SERVER_PORT` и
`LAIUE_SERVER_ALLOW_CONTENT_DOWNLOADS`.

## Проверка модов

До создания игрока сервер отправляет упорядоченный список обязательных модов.
Клиент сравнивает `id`, `version` и SHA-256 манифеста с entry DLL.

- лишний или отсутствующий `server`/`both` мод отклоняет подключение;
- `client`-моды в сравнении не участвуют;
- установленные, но выключенные моды можно включить через подтверждение UI;
- порядок модов является частью совместимости.

Только после точного ответа клиента сервер отправляет `SERVER_WELCOME` и
начинает принимать input. Стороны модов описаны в
[modding.md](modding.md).

## Загрузка содержимого

Если `allow_content_downloads = true`, сервер может предложить `.lmp`, `.lsp`
и `.ltp`. Загрузка начинается только после подтверждения пользователя.

Bundle ограничен 256 МиБ и 4096 файлами. Проверяются SHA-256, размеры, типы,
относительные пути и отсутствие `..`/reparse points. Распаковка идёт в
staging `*.download`, прежний пак сохраняется как `*.previous`, затем каталог
переключается и handshake повторяется.

SHA-256 подтверждает целостность полученного потока, но серверный `.lmp` всё
равно содержит нативный код с правами процесса.

## Авторитетность

| Состояние | Владелец и проверка |
|---|---|
| позиция и скорость | сервер; клиент их не присылает |
| движение | серверная fixed-step физика; клиент присылает кнопки и взгляд |
| ломание/установка | серверный raycast, дистанция, таймер, выбранный слот |
| инвентарь и дропы | сервер; bounded снимок 36 слотов |
| моды | точный ordered fingerprint до допуска |
| wire-поток | magic, version, exact size, sequence, state и rate limits |

Заголовок пакета имеет 16 байт, обычный payload ограничен 1024 байтами,
очереди и число peer ограничены. Неверный размер, NaN/Infinity, повтор
sequence, сообщение не в той фазе или overflow закрывают виновное соединение.

## Безопасность сборки

По решению проекта EXE/DLL собираются с `/GS-`, без CFG и CET compatibility;
ASLR и DEP/NX сохранены. Поэтому проверки входных данных нельзя ослаблять, а
текущий plaintext listener нельзя открывать наружу. Свойства «взломать
невозможно» не бывает: сервер уменьшает доверие к клиенту, но не устраняет
ошибки памяти, вредоносные моды, DDoS и уязвимости ОС.

## Что нужно для удалённой игры

1. MsQuic с TLS 1.3, проверкой сертификата и без gameplay в 0-RTT.
2. Identity/session tokens и управление ключами.
3. Initial snapshot, revisions/deltas чанков и interest management.
4. Сохранение игроков/инвентарей, метрики и административный аудит.
5. Fuzzing decoder, негативные сетевые проверки и нагрузочные прогоны.

До выполнения этих пунктов поддерживается только loopback.
