# Моддинг laiue

Моды живут в каталоге `mods/` рядом с `laiue.exe` и бывают двух видов:

- **декларативные** — текстовый файл `.lm`: переопределяет
  опубликованные параметры движка. Без компилятора, применяется вживую,
  битый файл просто не включится;
- **нативные** — каталог `.lmp` с DLL: полный доступ к API движка,
  свой код каждый кадр. Пишется и отлаживается **без исходников игры**
  — нужен только заголовок из `sdk/` рядом с исполняемым файлом.

Архитектурные решения — в
[content_architecture.md](content_architecture.md).

```text
mods/*.lm, *.lmp ──> mods/enabled.txt ──> .lm: реестр параметров
   (моды)             (что включено         .lmp: LoadLibrary + таблица
                       и в каком порядке)         функций LaiueModApi
```

## Первый мод за минуту

Создайте `mods/moon_walk.lm`:

```text
LAIUE MOD 1
name = Moon Walk
version = 1.0.0
game = 0.5

[gameplay]
extra_air_jumps = 3
air_jump_impulse = 5.0
```

Запустите игру → Esc → Настройки → Моды → включите тумблер.
Три дополнительных прыжка в воздухе работают сразу — применение живое.

## Формат файла

Построчный UTF-8. Первая значимая строка — заголовок формата, дальше
манифест (ключи до первой секции) и секции с параметрами:

```text
LAIUE MOD 1            # формат и его версия (сейчас 1)
name = Имя мода        # манифест: показывается в меню
author = Автор
version = 1.0.0        # версия мода (только для отображения)
game = 0.5             # минимальная версия игры (см. «Версии»)

[секция]
ключ = значение        # параметры из реестра
# строка-комментарий
```

Правила разбора: пробелы вокруг `=` игнорируются; регистр ключей не
важен; булевы значения — `true/false`, `1/0`, `on/off`, `yes/no`;
числа — десятичные с точкой. Неизвестные ключи и секции молча
пропускаются, поэтому мод для будущей версии игры не сломает старую.

## Реестр параметров (v1)

Значения зажимаются в диапазон. «Живое» применение означает — сразу
после щелчка тумблера, без перезапуска.

| Параметр | Тип | Диапазон | По умолчанию | Применение |
|---|---|---|---|---|
| `gameplay.extra_air_jumps` | целое | 0..3 | 0 | живое |
| `gameplay.air_jump_impulse` | число | 1..20 бл/с | 7.0 | живое |
| `gameplay.reset_on_ground` | булево | — | true | живое |
| `time.lock_enabled` | булево | — | false | живое |
| `time.locked_hour` | число | 0..24 | 12.0 | живое |
| `time.speed_preset` | `paused` \| `normal` \| `fast` \| `realtime` | — | не задан | живое |

Семантика: воздушные прыжки срабатывают отдельным нажатием прыжка в
полёте; `reset_on_ground = false` делает их одноразовыми до респавна.
`time.lock_enabled` держит солнце на `locked_hour`, что бы ни делала
скорость времени; `speed_preset` один раз переводит переключатель
скорости в админ-разделе.

Реестр расширяется только добавлением: существующие ключи не меняют
смысл и не удаляются внутри одной мажорной версии игры.

## Включение и порядок

Состоянием управляет `mods/enabled.txt` — по имени файла на строку:

```text
double_jump.lm
daylight_lock.lm
```

Файл — источник истины: вкладка «Моды» лишь редактирует его. Моды
применяются сверху вниз, и при конфликте одного параметра **побеждает
последний**. Включение из меню добавляет мод в конец списка. Файл можно
править руками — игра подхватит его при следующем запуске.

## Версии

Версия игры видна внизу меню паузы (`laiue 0.5.0`) и задаётся сборкой
(`project(laiue VERSION …)` → `LAIUE_VERSION_TEXT/MAJOR/MINOR`).

Строка `game = MAJOR.MINOR` в манифесте объявляет минимальную версию
игры. До 1.0 совместимость строгая: мажор должен совпасть, минор игры —
быть не меньше заявленного. Несовместимый мод виден в меню серым с
пометкой «нужна laiue X.Y» и не включается. Мод без строки `game`
считается совместимым с любой версией.

Версия формата в заголовке (`LAIUE MOD 1`) отвечает за сам синтаксис
файла и вырастет только при его несовместимом изменении.

## Встроенные примеры

**Double Jump** (`double_jump.lm`) — второй прыжок в воздухе: Space в
полёте даёт импульс `air_jump_impulse`, заряд возвращается при касании
земли. Использует все три параметра `gameplay`.

**Daylight Lock** (`daylight_lock.lm`) — вечное утро: время замирает на
10:30, переключатель скорости уходит в «Пауза». Демонстрирует секцию
`time` целиком.

Оба лежат в `assets/mods/` и копируются к исполняемому файлу при сборке.

## Нативные DLL-моды

Когда реестра параметров мало, мод пишется как обычная DLL на C.
Ключевое свойство: **мод не линкуется с игрой**. Все функции движка
приходят таблицей указателей при инициализации, поэтому не нужны ни
исходники, ни импортные библиотеки — только собранная игра и один
заголовок `sdk/laiue_mod_api.h` (лежит рядом с `laiue.exe`).

### Первый DLL-мод за пять минут

У вас есть каталог игры и установленная Visual Studio (любая, нужен
только компилятор). Создайте `my_mod.c`:

```c
#include "laiue_mod_api.h"

static const LaiueModApi* g_api;

static void OnFrame(void* user, float deltaSeconds)
{
    (void)user; (void)deltaSeconds;
    /* ваш код каждый кадр */
}

__declspec(dllexport) int32_t LaiueModInit(const LaiueModApi* api)
{
    g_api = api;
    api->setFrameCallback(api->host, OnFrame, 0);
    api->log(api->host, L"привет из мода!");
    return 0;
}

__declspec(dllexport) void LaiueModShutdown(void) {}
```

Сборка из «Developer Command Prompt for VS» (x64) в каталоге игры:

```bat
cl /nologo /W4 /O2 /utf-8 /LD /Isdk my_mod.c /Fe:my_mod.dll
```

(`/utf-8` важен, если в исходнике есть кириллические `L"строки"` —
иначе cl прочитает файл в системной кодировке и `log` выдаст мусор.)

Раскладка и включение:

```text
mods/
  my_mod.lmp/          <- каталог мода
    mod.lm             <- манифест (ниже)
    my_mod.dll
```

```text
LAIUE MOD 1
name = My Mod
version = 1.0.0
game = 0.5

[native]
entry = my_mod.dll
api = 1
```

Запустите игру → Esc → Настройки → Моды → тумблер. DLL загружается и
выгружается прямо по щелчку, без перезапуска игры.

### Контракт и API

Мод экспортирует `LaiueModInit(const LaiueModApi*)` (0 — успех; иное
отменяет загрузку) и опционально `LaiueModShutdown(void)`. Таблица v1:

| Функции | Что дают |
|---|---|
| `log` | строка в отладчик (OutputDebugString) и в `mods/mod_log.txt` |
| `getBlock`, `setBlock` | чтение и установка блоков; `setBlock` сам перестраивает чанки |
| `getPlayerPosition`, `setPlayerPosition` | позиция глаз (double, мир); телепорт обнуляет скорости |
| `getViewDirection`, `applyImpulse`, `isPlayerGrounded`, `getGameMode` | взгляд, толчок, опора, полёт/ходьба |
| `getTimeHours`, `setTimeHours` | время суток 0..24 |
| `setFrameCallback`, `setBlockEditCallback` | хук кадра и хук правки блока игроком (NULL — снять) |

Правила: все колбеки приходят на **главном потоке**, функции API
разрешено звать только из них; пока открыто меню паузы, кадровый хук
молчит; правки блоков самим модом событие `blockEdit` не порождают
(защита от рекурсии).

### Отладка без исходников игры

- Соберите мод с отладочной информацией: `cl /Zi /LD /Isdk my_mod.c`.
- Запустите игру и присоединитесь отладчиком: Visual Studio →
  Debug → Attach to Process → `laiue.exe` (или запустите
  `devenv /debugexe laiue.exe`). Брейкпоинты в вашем моде работают —
  PDB игры не нужен.
- `api->log()` виден в окне Output студии, в DebugView и копится в
  `mods/mod_log.txt` — там же строки «мод загружен/выгружается».
- DLL занята, пока мод включён: перед пересборкой выключите его
  тумблером (хост сделает FreeLibrary), пересоберите и включите снова —
  быстрый цикл правок без перезапуска игры.

### Производительность

- Кадровый хук зовётся каждый кадр (60+ раз в секунду): без аллокаций
  и файлового ввода-вывода в нём; тяжёлое состояние держите в статике.
- `setBlock` перестраивает затронутые чанки в рабочих потоках — это
  дёшево поштучно, но не заливайте тысячи блоков за кадр: ремешинг
  имеет бюджет, хвост достроится в следующих кадрах.
- Собирайте с `/O2`; CRT можно использовать любую (`/MT` удобнее —
  без внешних зависимостей DLL); сама игра без CRT, ей всё равно.
- `log` — не для каждого кадра: он пишет в файл.

### Версии и совместимость

`api = N` в манифесте — версия таблицы API из SDK; игра отклоняет мод,
если её версия ниже. Таблица растёт только добавлением полей в хвост:
проверяйте `structSize`, прежде чем звать функции новее вашей копии
SDK. `game = MAJOR.MINOR` работает как у декларативных модов.
Несовместимый мод виден в меню серым и не грузится.

### Встроенный пример

`mods/auto_bridge.lmp` — «мост под ногами»: в режиме ходьбы, пока
игрок не на земле, подкладывает блок под ступни. Исходник, манифест и
`build.bat` — в `sdk/examples/auto_bridge/`; собирается и внешним
`cl /LD`, и вместе с игрой (цель `mod_auto_bridge`).
