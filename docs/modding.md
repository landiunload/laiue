# Моддинг laiue

Мод laiue — это нативная DLL. Каталог `mods/<имя>.lmp` рядом с
исполняемым файлом содержит манифест `mod.lm` и саму DLL. Клиентский
состав хранится в `mods/enabled.txt`, серверный — в
`mods/server_enabled.txt`.

Ключевое свойство: **мод не линкуется с игрой**. Все функции движка
приходят таблицей указателей при инициализации, поэтому для разработки
не нужны ни исходники, ни импортные библиотеки — только собранная игра
и один заголовок `sdk/laiue_mod_api.h` (лежит рядом с исполняемым
файлом). Архитектурные решения — в
[content_architecture.md](content_architecture.md).

```text
mods/*.lmp ──> mods/enabled.txt ──> ModHost: LoadLibrary + таблица
 (манифест      (что включено        функций LaiueModApi, хуки кадра
  mod.lm + DLL)  и в каком порядке)   и фиксированного тика, интерфейсы
                                      между модами
```

## Первый мод за пять минут

Нужны каталог игры и компилятор Visual Studio. Создайте `my_mod.c`:

```c
#include "laiue_mod_api.h"

static const LaiueModApi* g_api;

static void OnFixedTick(void* user, float stepSeconds)
{
    (void)user; (void)stepSeconds;
    /* геймплей: шаг всегда 1/60 с, FPS не влияет */
}

__declspec(dllexport) int32_t LaiueModInit(const LaiueModApi* api)
{
    g_api = api;
    api->setFixedTickCallback(api->host, OnFixedTick, 0);
    api->log(api->host, L"привет из мода!");
    return 0;
}

__declspec(dllexport) void LaiueModShutdown(void) {}
```

Сборка из «Developer Command Prompt for VS» (x64) в каталоге игры:

```bat
cl /nologo /W4 /O2 /utf-8 /LD /Isdk my_mod.c /Fe:my_mod.dll
```

(`/utf-8` обязателен при кириллице в `L"строках"` — иначе cl прочитает
файл в системной кодировке и `log` выдаст мусор.)

Раскладка и манифест:

```text
mods/
  my_mod.lmp/
    mod.lm
    my_mod.dll
```

```text
LAIUE MOD 1
id = example.my_mod
name = My Mod
version = 1.0.0
game = 0.5
side = both

[native]
entry = my_mod.dll
api = 1
```

Запустите игру → Esc → Настройки → Моды → тумблер. DLL загружается и
выгружается по щелчку, без перезапуска игры.

## Стороны и подключение к серверу

`id` — стабильный ASCII-идентификатор (`a-z`, цифры, `.`, `_`, `-`), а
`side` принимает одно из трёх значений:

| `side` | Где исполняется | Участие в сетевой совместимости |
|---|---|---|
| `client` | только клиент | не участвует; таких модов может быть сколько угодно |
| `server` | только dedicated server | обязан быть установлен у клиента, но там не загружается |
| `both` | клиент и сервер | обязан совпасть и загружается на обеих сторонах |

Если `side` не указан, для обратной совместимости используется `both`;
неизвестное значение делает манифест несовместимым. Сервер до допуска игрока
присылает свой упорядоченный набор. Сравниваются `id`, `version` и полный
SHA-256 манифеста с DLL. Лишний включённый `server`/`both` мод тоже является
несовпадением. Если нужные версии уже установлены, клиент предлагает включить
их в серверном порядке и отключить лишние; включённые `client`-моды при этом
сохраняются.

Сервер загружает `server` и `both` из `server_enabled.txt`. Клиент загружает
`client` и `both` из `enabled.txt`; мод не своей стороны получает состояние
«неактивен на этой стороне», а не ошибку DLL.

## Контракт мода

Мод экспортирует две функции:

- `int32_t LaiueModInit(const LaiueModApi* api)` — один раз при
  включении (главный поток, мир уже создан, функции api можно звать
  прямо отсюда). 0 — успех; любой другой код отменяет загрузку, попадает
  в журнал и показывается на вкладке модов («отказ Init, код N»).
- `void LaiueModShutdown(void)` — опционально, при выключении и выходе.
  Снимать колбеки не обязательно: хост делает это сам до FreeLibrary.

Смена состава включённых модов **перезагружает всю цепочку** в порядке
соответствующего enabled-файла — состояние модов при этом сбрасывается, зато между модами
не бывает висячих указателей.

## API (версия 1)

| Функции | Что дают |
|---|---|
| `log` | строка в отладчик (OutputDebugString) и в `mods/mod_log.txt` |
| `getBlock`, `setBlock` | чтение и установка блоков; `setBlock` сам перестраивает чанки |
| `getPlayerPosition`, `setPlayerPosition` | позиция глаз (double, мир); телепорт обнуляет скорости |
| `getViewDirection`, `applyImpulse`, `isPlayerGrounded`, `getGameMode` | взгляд, толчок, опора, креатив/выживание |
| `getTimeHours`, `setTimeHours` | время суток 0..24 |
| `setAirJumps` | воздушные прыжки (0..3, импульс, восстановление на земле) |
| `setFrameCallback` | хук кадра — **визуал**: приходит render-`deltaSeconds` |
| `setFixedTickCallback` | хук фиксированного тика — **геймплей**: шаг всегда 1/60 с, не больше 5 тиков за кадр |
| `setBlockEditCallback` | правка блока игроком (изменения самого мода событие не порождают) |
| `publishInterface`, `queryInterface` | библиотечные моды, см. ниже |
| `writeModData`, `readModData` | блоб мода в сохранении мира: читайте в Init, пишите в Shutdown (см. [world_format.md](world_format.md)) |

Правила: все колбеки приходят на главном потоке, функции API разрешено
звать только из них и из `LaiueModInit`; пока открыто меню паузы, хуки
молчат. Совместимость: таблица растёт только добавлением полей в хвост —
проверяйте `structSize`, прежде чем звать функции новее вашей копии SDK.

## Библиотечные моды

Мод может быть библиотекой: он публикует именованную таблицу функций,
а другие моды получают её по имени. Так «мод с крутыми функциями»
становится зависимостью для чужих модов.

**Автор библиотеки** распространяет два артефакта: свой `.lmp` и
заголовок с интерфейсом. Публикация — в `LaiueModInit`:

```c
/* builder_lib.h — этот файл вы отдаёте другим моддерам */
#define BUILDER_LIB_NAME "laiue.builder"
#define BUILDER_LIB_VERSION 1u
typedef struct BuilderLibV1
{
    uint32_t version;
    void (*fillBox)(int64_t x0, int64_t y0, int64_t z0,
        int64_t x1, int64_t y1, int64_t z1, uint8_t block);
    void (*buildHelix)(int64_t centerX, int64_t centerY, int64_t baseZ,
        int32_t radius, int32_t height, uint8_t block);
} BuilderLibV1;

/* в библиотеке */
static BuilderLibV1 g_builder = { BUILDER_LIB_VERSION, FillBox, BuildHelix };
api->publishInterface(api->host,
    BUILDER_LIB_NAME, BUILDER_LIB_VERSION, &g_builder);
```

**Потребитель** запрашивает интерфейс и честно отказывается, если его
нет — причина попадёт в журнал и на вкладку модов:

```c
const BuilderLibV1* builder = api->queryInterface(api->host,
    BUILDER_LIB_NAME, BUILDER_LIB_VERSION);
if (builder == NULL)
{
    api->log(api->host, L"нужен Builder Lib выше по списку");
    return 1;
}
builder->buildHelix(0, 18, 98, 4, 26, LAIUE_BLOCK_GRASS);
```

Правила:

- **порядок**: библиотека стоит в `enabled.txt` (и включается) раньше
  потребителя — интерфейсы публикуются в порядке загрузки;
- **версии**: `version` растёт при добавлении функций в хвост таблицы;
  ломающие изменения — новое имя (`"laiue.builder2"`);
  `queryInterface` вернёт NULL, если версия ниже запрошенной;
- **время жизни**: указатель принадлежит библиотеке; хост перезагружает
  цепочку целиком при любой смене состава, поэтому указатель нельзя
  пережить — просто получайте его заново в `LaiueModInit`;
- одно имя — один владелец: повторная публикация чужого имени
  отвергается.

Живой пример пары — `builder_lib.lmp` + `spiral_tower.lmp` в
`sdk/examples/`: библиотека публикует `laiue.builder`, потребитель
строит через неё площадку и спиральную башню.

## Статусы на вкладке модов

Тумблер показывает желание, а подпись — факт от хоста:

| Подпись | Причина |
|---|---|
| — | мод загружен и работает |
| `нужна laiue X.Y` | версия игры ниже `game =` из манифеста |
| `манифест неполон` | нет `[native] entry`/`api` или api новее игры |
| `DLL не загрузилась` | нет файла DLL или экспорта `LaiueModInit` |
| `отказ Init, код N` | `LaiueModInit` вернул N (типично — нет библиотеки) |

## Отладка без исходников игры

- Соберите с отладочной информацией: `cl /Zi /utf-8 /LD /Isdk my_mod.c`.
- Присоединитесь отладчиком: Visual Studio → Debug → Attach to Process →
  `laiue.exe` (или `devenv /debugexe laiue.exe`). Брейкпоинты в вашем
  моде работают — PDB игры не нужен.
- `api->log()` виден в Output студии, в DebugView и копится в
  `mods/mod_log.txt` вместе со строками «мод загружен/выгружается».
- DLL занята, пока мод включён: перед пересборкой выключите его
  тумблером, пересоберите, включите снова — быстрый цикл без
  перезапуска игры.

## Производительность

- Геймплей — в `setFixedTickCallback` (детерминированный шаг), визуал —
  в `setFrameCallback`; в обоих — без аллокаций и файлового
  ввода-вывода, `log` не для каждого кадра.
- `setBlock` перестраивает чанки в рабочих потоках — поштучно дёшево,
  но тысячи блоков за кадр упрутся в бюджет ремешинга (хвост достроится
  в следующих кадрах).
- Собирайте с `/O2`; CRT любая (`/MT` удобнее — DLL без зависимостей);
  сама игра без CRT, ей всё равно.

## Версии

Версия игры видна внизу меню паузы и приходит моду в
`gameVersionMajor/Minor`. `game = MAJOR.MINOR` в манифесте — минимальная
версия игры (до 1.0 строго: мажор совпадает, минор не меньше).
`api = N` — версия таблицы SDK; игра отклоняет мод, если её версия ниже.
`LAIUE MOD 1` в заголовке — версия синтаксиса манифеста.

## Встроенные примеры (`sdk/examples/`)

| Мод | Показывает |
|---|---|
| `double_jump` (`client`) | геймплейная функция API одним вызовом |
| `daylight_lock` (`server`) | серверный хук времени |
| `auto_bridge` (`client`) | локальная правка мира по положению игрока |
| `builder_lib` (`server`) | библиотека: публикация интерфейса `laiue.builder` |
| `spiral_tower` (`server`) | потребитель библиотеки и честный отказ без неё |

Каждый собирается и внешним `cl /LD` (см. `build.bat`), и вместе с
игрой (CMake-цели `mod_*`). DLL-мод — полное доверие, как в любой игре
с нативным моддингом: ставьте только моды, чьим авторам и серверу вы
доверяете. Серверная загрузка никогда не запускается молча: пользователь
подтверждает её в меню, поток проверяется SHA-256 и ставится через staging.
Это защищает целостность передачи, но не превращает полученную DLL в песочницу.

## Сохранения

Мир сохраняется автоматически при выходе и кнопкой «Сохранить мир»
(Настройки → Админ): seed, правки блоков, время суток, позиция игрока,
`mods.lock` с составом модов и данные модов — формат описан в
[world_format.md](world_format.md). `LaiueModShutdown` вызывается до
записи сохранения, поэтому запись данных мода в нём гарантированно
попадает в сейв.
