# Форматы содержимого laiue

Все пользовательское содержимое разделено по назначению. Одиночная сущность
имеет двухбуквенное расширение, контейнер-пак — соответствующее расширение с
суффиксом `p`.

| Назначение | Одиночный формат | Пак | Каталог runtime |
|---|---|---|---|
| Универсальные ресурсы | `.lr` | `.lrp` | `resources/` |
| Моды | `.lm` | `.lmp` | `mods/` |
| Шейдеры | `.ls` | `.lsp` | `shaders/` |
| Данные | `.ld` | `.ldp` | `data/` |
| Текстуры | `.lt` | `.ltp` | `textures/` |

## Правила

- Форматы не являются взаимозаменяемыми: `.lrp` не считается текстурпаком.
- Активный пак категории хранится в UTF-8-файле `active.txt` внутри её
  каталога.
- Имена проверяются до построения пути: запрещены абсолютные пути,
  `..`, разделители каталогов и зарезервированные Windows-символы.
- Каталог может перечисляться через `LaiueContentEnumerate`; будущие системы
  модов, данных и ресурсов используют тот же API, а не собственные копии
  Win32-поиска.

## Шейдеры

Шейдерпак — каталог с расширением `.lsp`. Его отдельные байткоды имеют
расширение `.ls`:

```text
shaders/
  cinematic.lsp/
    chunk_vs.ls
    chunk_ps.ls
    panorama_vs.ls
    panorama_ps.ls
    ui_vs.ls
    ui_ps.ls
```

## Текстуры

`.ltp` остаётся бинарным текстурпаком LTP. Одиночный `.lt` зарезервирован для
будущего адресуемого текстурного ресурса. Текстурпаки находятся в
`textures/`, а не в `texturepacks/`.
