# Форматы содержимого laiue

> Здесь описано текущее состояние. Целевая архитектура и принятые
> решения (какие категории остаются, модель модов, контракт
> шейдерпаков) — в [content_architecture.md](content_architecture.md).
> Моды — только DLL-паки `.lmp` (одиночный `.lm` служит манифестом
> внутри пака); формат, API и `mods/enabled.txt` — в
> [modding.md](modding.md).

Все пользовательское содержимое разделено по назначению. Одиночная сущность
имеет двухбуквенное расширение, контейнер-пак — соответствующее расширение с
суффиксом `p`.

| Назначение | Одиночный формат | Пак | Каталог runtime |
|---|---|---|---|
| Моды | `.lm` | `.lmp` | `mods/` |
| Шейдеры | `.ls` | `.lsp` | `shaders/` |
| Текстуры | `.lt` | `.ltp` | `textures/` |

## Правила

- Форматы не являются взаимозаменяемыми: `.lsp` не считается текстурпаком.
- Активный пак категории хранится в UTF-8-файле `active.txt` внутри её
  каталога.
- Имена проверяются до построения пути: запрещены абсолютные пути,
  `..`, разделители каталогов и зарезервированные Windows-символы.
- Каталог перечисляется через `LaiueContentEnumerate`; системы модов,
  шейдеров и текстур используют общий API, а не собственные копии
  Win32-поиска.
- `.lr/.lrp` и `.ld/.ldp` удалены: у них не было отдельной семантики и
  runtime-потребителя. Данные конкретной системы принадлежат этой системе
  или нативному моду.

## Шейдеры

Шейдерпак — каталог с расширением `.lsp`. Его отдельные байткоды имеют
расширение `.ls`:

```text
shaders/
  cinematic.lsp/
    chunk_vs.ls
    chunk_ps.ls
    panorama_vs.ls
    panorama_ps.ls
    ui_vs.ls
    ui_ps.ls
```

В корне обязателен `pack.lm` с версией контракта. Стадии могут быть
частичными; полный контракт описан в [shaderpacks.md](shaderpacks.md).

## Текстуры

`.ltp` остаётся бинарным текстурпаком LTP. Одиночный `.lt` зарезервирован для
будущего адресуемого текстурного ресурса. Текстурпаки находятся в
`textures/`, а не в `texturepacks/`.

## Схема манифеста `.lm`

Манифест мода — построчный UTF-8-текст. Первая строка —
`LAIUE MOD <версия>`. Далее идут строки `ключ = значение`; строки,
начинающиеся с `#`, — комментарии. Полный исполняемый контракт описан в
[modding.md](modding.md).

```text
LAIUE MOD 1
name = Double Jump
version = 1.0.0
[gameplay]
extra_air_jumps = 1
```

## Встроенное тестовое содержимое

Сборка кладёт рядом с `laiue.exe` по два примера каждого вида:

| Каталог | Примеры |
|---|---|
| `mods/` | пять исполняемых примеров SDK в каталогах `.lmp` |
| `shaders/` | `photon.lsp`, `vibrant.lsp` |
| `textures/` | `laiue_32x.ltp`, `patrix_32x.ltp`, `laiue_soft_16x.ltp` |

### Шейдерпаки

`photon.lsp` — атмосферный рендер в духе Photon: тёплое солнце и холодный
небесный ambient, мягкая небесная окклюзия, экспоненциальный туман под цвет
неба, filmic-тонемаппинг ACES и обводка солнцем по контуру. `vibrant.lsp` —
лёгкий чистый вид без тумана с усиленной насыщенностью и контрастным
терминатором. Оба совместимы со встроенным cbuffer: гамма из настроек
приходит в шейдер отдельным компонентом. Активный пак применяется сразу при
старте, а не только по кнопке в меню.
